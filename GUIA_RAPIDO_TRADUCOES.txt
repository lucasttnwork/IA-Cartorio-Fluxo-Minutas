================================================================================
                      DICAS DE IMPLEMENTAÇÃO
================================================================================

1. USE i18next
   - Biblioteca mais popular e bem documentada
   - Suporta lazy loading de traduções
   - Ferramenta web de gerenciamento (i18next.com)

2. ESTRUTURE POR CONTEXTO
   - auth.json para autenticação
   - dashboard.json para dashboard
   - cases.json para operações de casos
   - entities.json para entidades
   - flow.json para fluxo de compra/venda
   - errors.json para mensagens de erro

3. USE KEYS DESCRITIVAS
   ✓ BOAS:   t('dashboard.page.title')
   ✗ RUINS:  t('title1'), t('btn_ok')

4. MANTENHA CONSISTÊNCIA
   - Use mesmos termos em toda a aplicação
   - Crie glossário de termos notariais
   - Revise com especialista em direito

5. INTERNACIONALIZAÇÃO FUTURA
   - Estruture para suportar múltiplos idiomas
   - Não codifique strings em português
   - Use sempre `t()` hooks em componentes
